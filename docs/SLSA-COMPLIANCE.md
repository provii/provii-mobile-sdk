# SLSA Compliance

The provii-mobile-sdk implements [SLSA](https://slsa.dev) Level 3 supply chain security for all release artifacts.

## Build Requirements

| SLSA Requirement | Implementation |
|-----------------|----------------|
| Build as code | GitHub Actions workflows in `.github/workflows/release.yml` |
| Hermetic build | `--locked` flag ensures exact `Cargo.lock` dependency versions |
| Ephemeral environment | GitHub-hosted runners; no persistent build state |
| Isolated build | Each job runs in a fresh container or VM |
| Parameterless | Release triggered by semver tag push only |
| Scripted build | Fully automated, no manual steps |

## Provenance

Every release includes a SLSA provenance attestation (`provii-mobile-sdk.intoto.jsonl`) generated by the [SLSA GitHub Generator](https://github.com/slsa-framework/slsa-github-generator). The attestation records the build inputs, builder identity, and output hashes.

## Verification

```bash
# Verify SLSA provenance
slsa-verifier verify-artifact \
  provii-mobile-sdk-bundle.tar.gz \
  --provenance-path provii-mobile-sdk.intoto.jsonl \
  --source-uri github.com/provii/provii-mobile-sdk
```

## Signing

All release artifacts are signed with [Sigstore](https://sigstore.dev) keyless signing. The signing identity is bound to the GitHub Actions OIDC token, so signatures can be verified without managing public keys.

```bash
cosign verify-blob \
  --bundle provii-mobile-sdk-bundle.cosign-bundle \
  --certificate-identity-regexp="https://github.com/provii/provii-mobile-sdk/.*" \
  --certificate-oidc-issuer="https://token.actions.githubusercontent.com" \
  provii-mobile-sdk-bundle.tar.gz
```

## Audit Trail

| Artifact | Purpose |
|----------|---------|
| `provii-mobile-sdk-bundle.tar.gz.sha256` | SHA-256 checksum for integrity verification |
| `provii-mobile-sdk-bundle.cosign-bundle` | Sigstore signature bundle (certificate + signature) |
| `provii-mobile-sdk.intoto.jsonl` | SLSA v1.0 provenance attestation (in-toto format) |
| `provii-mobile-sdk-sbom.cyclonedx.json` | CycloneDX 1.6 software bill of materials |
| `provii-mobile-sdk-sbom.spdx.json` | SPDX 2.3 software bill of materials |

## Dependency Auditing

Automated `cargo audit` runs on every CI build and release. The `deny.toml` configuration enforces licence allowlists and blocks unknown registries. Only crates from crates.io and the `provii` GitHub organisation are permitted.
